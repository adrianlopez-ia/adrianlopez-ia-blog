# House Bot: Automatizando la Busqueda de Vivienda en Madrid con IA

Buscar vivienda en Madrid es una tarea agotadora. Decenas de cooperativas, constructoras con webs desactualizadas, formularios de contacto interminables y la sensacion constante de que te estas perdiendo algo. Asi que hice lo que cualquier desarrollador haria: **automatizarlo todo**.

[**house-bot**](https://github.com/adrianlopez-ia/house-bot) es un bot en Python que descubre webs de cooperativas y constructoras, analiza las oportunidades con IA, rellena formularios de contacto automaticamente y me envia alertas por Telegram. Todo gratis.

House Bot is a specialized AI agent designed to navigate the complex Madrid housing market. By leveraging a **Multi-Provider AI Pool** (switching between OpenRouter and Groc), it ensures 24/7 availability even when API limits are reached.

## El Problema

El mercado de vivienda nueva en Madrid (especialmente cooperativas en zonas norte, este y oeste) tiene varios problemas:

- **Informacion dispersa**: cada cooperativa tiene su propia web, muchas con informacion pobre
- **Formularios manuales**: para obtener info detallada hay que rellenar formularios de contacto en cada una
- **Sin agregadores fiables**: los portales inmobiliarios se centran en segunda mano
- **Ventanas de oportunidad cortas**: las mejores cooperativas se llenan rapido

## Stack Tecnologico

## Como Funciona Cada Modulo

### 1. Discovery: Encontrando Cooperativas

El `DiscoveryService` combina dos fuentes para encontrar webs de cooperativas:

- **Lista semilla**: sitios conocidos de cooperativas y constructoras en Madrid
- **Busqueda activa**: queries automaticas en DuckDuckGo para descubrir nuevos sitios

```python
# Las queries se generan automaticamente para cada zona
# "cooperativa vivienda Madrid norte"
# "constructora pisos nuevos Madrid este"
# "promocion obra nueva Madrid oeste"
```

Cada sitio descubierto se almacena en SQLite con su estado (activo, analizado, descartado).

### 2. AI Analysis: Gemini Analiza Cada Web

El `ScraperService` utiliza Playwright para navegar cada web, extraer el contenido y enviarlo a Gemini AI para su analisis:

      AI-->>S: JSON estructurado
      S->>DB: Guardar oportunidades

  caption="Flujo de analisis con IA"
/>

Gemini devuelve datos estructurados: nombre del proyecto, zona, rango de precios, plazos de entrega, si hay formulario de contacto, y una **puntuacion de interes** del 1 al 10.

La capa de IA esta abstraida con un `Protocol`, lo que permite cambiar de proveedor sin modificar nada:

```python
class AIAnalyzer(Protocol):
    async def analyze_site(self, url: str, content: str) -> list[Opportunity]: ...
    async def analyze_form(self, url: str, html: str) -> FormAnalysis: ...
```

Se soportan multiples proveedores gratuitos: **Cerebras**, **Groc**, **Mistral**, **xAI** y **Gemini**. Actualmente, el blog implementa un **AI Pool Dinámico con Triple Failover**: el sistema intenta primero conectar con **OpenRouter**, si falla o alcanza su límite, salta instantáneamente a **Groc**, y como última capa de redundancia utiliza **Google Gemini**. Esto garantiza una disponibilidad total del servicio sin interrupciones.

### 3. Form Filling: Formularios Automaticos

El `FormService` detecta formularios de contacto en las webs y los rellena automaticamente con tus datos personales:

1. **Deteccion**: funciones puras analizan el HTML buscando campos de formulario
2. **Mapeo**: cada campo se asocia a un dato personal (nombre, email, telefono...)
3. **Rellenado**: Playwright simula la interaccion humana para rellenar y enviar
4. **Evidencia**: se toma un screenshot de cada formulario enviado

```python
# Configuracion de datos personales en .env
USER_FULL_NAME=Tu Nombre
USER_EMAIL=tu@email.com
USER_PHONE=+34600000000
```

### 4. Telegram Bot: Control Remoto

El bot de Telegram permite monitorizar y controlar house-bot desde el movil:

| Comando | Que hace |
|---------|----------|
| `/oportunidades` | Lista de oportunidades activas con puntuacion |
| `/futuras` | Proximos lanzamientos detectados |
| `/formularios` | Estado de formularios enviados y pendientes |
| `/buscar` | Forzar busqueda inmediata |
| `/reporte` | Resumen completo con estadisticas |
| `/sites` | Webs monitorizadas |

### 5. Web Dashboard: Vista Completa

House-bot incluye un dashboard web completo con:

- **Estadisticas en tiempo real** via Server-Sent Events (SSE)
- **Lista de oportunidades** filtrable por zona y estado
- **Gestion de formularios** con evidencia de envio
- **Control del AI provider** para cambiar de modelo en caliente
- **Acciones manuales**: lanzar busqueda, scraping o rellenado de formularios

El dashboard se sirve como SPA estatica desde aiohttp, con una API REST completa.

> Puedes probar el dashboard en la seccion [Apps](/apps/house-bot) de este blog.

Lo que hace este proyecto interesante desde el punto de vista de ingenieria:

- **Dependency Injection**: todas las dependencias se inyectan via constructor. `main.py` es el unico lugar donde se cablea todo.
- **Protocol-based abstractions**: el proveedor de IA se puede cambiar sin tocar el resto del codigo.
- **Repository pattern**: acceso a datos centralizado con async context manager.
- **Frozen dataclasses + Enums**: modelos inmutables, sin magic strings.
- **Validated config**: Pydantic Settings valida todo al arrancar.
