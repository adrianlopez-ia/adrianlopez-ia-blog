# House Bot: Automating Madrid Housing Search with AI

Finding a home in Madrid is an exhausting task. Dozens of cooperatives, construction companies with outdated websites, endless contact forms, and the constant feeling that you're missing something. So I did what any developer would do: **automate everything**.

[**house-bot**](https://github.com/adrianlopez-ia/house-bot) is a Python bot that discovers cooperative and developer websites, analyzes opportunities with AI, fills out contact forms automatically, and sends me alerts via Telegram. All for free.

House Bot is a specialized AI agent designed to navigate the complex Madrid housing market. By leveraging a **Multi-Provider AI Pool** (switching between OpenRouter and Groq), it ensures 24/7 availability even when API limits are reached.

## The Problem

The new housing market in Madrid (especially cooperatives in northern, eastern, and western zones) faces several issues:

- **Scattered Information**: each cooperative has its own website, many with poor information.
- **Manual Forms**: to get detailed info, you must fill out contact forms on each one.
- **No Reliable Aggregators**: real estate portals focus on second-hand properties.
- **Short Windows of Opportunity**: the best cooperatives fill up fast.

## How Each Module Works

### 1. Discovery: Finding Cooperatives

The `DiscoveryService` combines two sources to find cooperative websites:

- **Seed List**: known cooperative and developer sites in Madrid.
- **Active Search**: automated DuckDuckGo queries to discover new sites.

```python
# Queries are generated automatically for each zone
# "housing cooperative Madrid north"
# "new apartment construction Madrid east"
# "new build development Madrid west"
```

Each discovered site is stored in SQLite with its status (active, analyzed, discarded).

### 2. AI Analysis: Gemini Analyzes Each Web

The `ScraperService` uses Playwright to navigate each site, extract content, and send it to Gemini AI for analysis:

Gemini returns structured data: project name, zone, price range, delivery deadlines, whether there is a contact form, and an **interest score** from 1 to 10.

The AI layer is abstracted with a `Protocol`, allowing for provider changes without modification:

```python
class AIAnalyzer(Protocol):
    async def analyze_site(self, url: str, content: str) -> list[Opportunity]: ...
    async def analyze_form(self, url: str, html: str) -> FormAnalysis: ...
```

Multiple free providers are supported: **Cerebras**, **Groq**, **Mistral**, **xAI**, and **Gemini**. Currently, the blog implements a **Dynamic AI Pool with Triple Failover**: the system first tries to connect with **OpenRouter**, if it fails or reaches its limit, it instantly jumps to **Groq**, and as a final redundancy layer, it uses **Google Gemini**. This ensures total service availability without interruptions.

### 3. Form Filling: Automatic Forms

The `FormService` detects contact forms on websites and fills them automatically with your personal data:

1. **Detection**: pure functions analyze the HTML looking for form fields.
2. **Mapping**: each field is associated with personal data (name, email, phone...).
3. **Filling**: Playwright simulates human interaction to fill and submit.
4. **Evidence**: a screenshot is taken for each submitted form.

```python
# Personal data configuration in .env
USER_FULL_NAME=Your Name
USER_EMAIL=your@email.com
USER_PHONE=+34600000000
```

### 4. Telegram Bot: Remote Control

The Telegram bot allows monitoring and controlling house-bot from your mobile:

| Command | Action |
|---------|----------|
| `/opportunities` | List of active opportunities with scores |
| `/future` | Detected upcoming launches |
| `/forms` | Status of sent and pending forms |
| `/search` | Force immediate search |
| `/report` | Full summary with statistics |
| `/sites` | Monitored websites |

### 5. Web Dashboard: Complete View

House-bot includes a full web dashboard with:

- **Real-time Statistics** via Server-Sent Events (SSE)
- **Opportunity List** filterable by zone and status
- **Form Management** with submission evidence
- **AI Provider Control** to switch models on the fly
- **Manual Actions**: launch search, scraping, or form filling

The dashboard is served as a static SPA from aiohttp, with a full REST API.

- **Dependency Injection**: all dependencies are injected via constructor. `main.py` is the only place where everything is wired.
- **Protocol-based abstractions**: the AI provider can be changed without touching the rest of the code.
- **Repository pattern**: centralized data access with async context manager.
- **Frozen dataclasses + Enums**: immutable models, no magic strings.
- **Validated config**: Pydantic Settings validates everything at startup.
